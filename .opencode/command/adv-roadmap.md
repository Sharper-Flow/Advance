# ADV Roadmap

Show the prioritized backlog: bugs by priority, features ranked by WSJF, and any items deferred (missing Value or kind). Read-only. Source of truth is the GitHub Project board; ROADMAP.md and `.adv/roadmap-snapshot.json` are deterministic mirrors regenerated by `/adv-triage`.

> **rq-backlogCoord01..05 note:** When Temporal Visibility is reachable, this command internally resolves active-change annotations via a single `AdvBacklogIssueNumber IN (...)` query (O(1)) ŌĆö same data path used by the agent-canonical `adv_backlog_state` tool. The legacy O(n├Śm) `store.changes.get` loop is the fallback when Visibility is unavailable. New code should prefer `adv_backlog_state` directly for the typed read interface + TTL-bounded freshness contract.

> **CHECKLIST**: Read-only. Two source modes ŌĆö file (default, fast, deterministic) and live (always fresh, costs ~1 GraphQL point). Cross-references active ADV changes via `change.origin.issue_number` so users see what's already in flight.

## Parse Flags

Extract from `$ARGUMENTS`:

- `--live` ŌĆö query GitHub Project directly instead of reading `.adv/roadmap-snapshot.json`. Use when file may be stale (no recent `/adv-triage` run) or when verifying a just-edited Project field.
- `--top <N>` ŌĆö limit features to top N by WSJF. Default: 10.
- `--kind {bug|feature|all}` ŌĆö filter by item kind. Default: `all`.
- `--priority {critical|high|medium|low}` ŌĆö filter bugs to a single tier. No effect on features.

Invalid flags or values reject with a single-line error and the valid list.

## Phase 1: Read backlog state

Call `adv_roadmap` with the parsed args:

```
adv_roadmap({
  source: <"file" | "live">,
  kind: <"bug" | "feature" | "all">,
  top: <N>,
  priority: <"critical" | "high" | "medium" | "low">,
})
```

| Source | Behavior |
|---|---|
| `file` (default) | Reads `.adv/roadmap-snapshot.json`. Returns actionable error if missing ŌĆö recommend `/adv-triage` to regenerate. Emits freshness metadata and warnings after 2h so post-ATC/archive churn cannot look like untouched backlog. |
| `live` | Calls `gh project item-list <N> --owner <owner>` (appends `--query "repo:<owner>/<repository_filter>"` server-side when the typed config sets `repository_filter` ŌĆö rq-repoFilter01). Requires `github_project` metadata persisted (run `/adv-triage` once to bootstrap). |

The tool also walks active ADV changes (`status Ōłł {draft, pending, active}`) and annotates roadmap items with their in-flight change ID via `change.origin.issue_number` lookup. Items already being worked carry `active_change: <id>` in the response.

## Phase 2: Render to chat (mandatory)

Emit the ranked backlog as inline tables. Cross-referenced items show their active change in the right-most column. The render is the agent's primary output ŌĆö do NOT replace it with a "see ROADMAP.md" pointer.

### Layout

```markdown
## Roadmap (source: {file|live}, generated: {ISO-8601})

Project: #{N} ({owner}/{title})
Freshness: {freshness.status}; age {freshness.age_hours}h; refresh after {freshness.stale_after_hours}h
Total: {bugs}/{features}/{deferred} (bugs/features/deferred), {active_changes_indexed} active change(s) cross-referenced

Warnings:
- {warnings[]}

### Bugs ŌĆö {priority tier}
| # | Title | Active change |
|---|-------|---------------|
| #{n} | {title} | {change-id or `ŌĆö`} |

(Skip empty tiers. Repeat for each tier present in the result.)

### Features (top {N} by WSJF)
| # | Title | V | TC | RROE | E | WSJF | Active change |
|---|-------|---|----|------|---|------|---------------|
| #{n} | {title} | {V} | {TC} | {RROE} | {E} | {WSJF} | {change-id or `ŌĆö`} |

### Deferred / Unscored ({count})
- #{n} ŌĆö {title} ŌĆö _reason_
```

If `kind=feature`, drop the bugs section. If `kind=bug`, drop the features section. If `priority` is set, only show that bug tier.

Omit the Warnings block when `warnings` is empty. If `freshness.needs_refresh === true`, render the freshness line and warnings before recommendations. Do not recommend starting `/adv-proposal #N` from a stale file snapshot until user has checked `/adv-roadmap --live` or refreshed via `/adv-triage`.

### Recommendations

After the tables, surface 1ŌĆō3 actionable next steps based on what was found:

| Found | Recommendation |
|---|---|
| Top feature has no `active_change` | `/adv-proposal #{n}` |
| Critical bug has no `active_change` | `/adv-proposal #{n}` |
| Item already in flight (`active_change` present) | `/adv-status` (or named `change-id`) to inspect progress |
| Snapshot is older than 2 hours (file source) or `freshness.needs_refresh` | `/adv-roadmap --live` before acting; `/adv-triage` to refresh mirror |
| Many deferred features (Ōēź3) | `/adv-triage` and use `autofill` for the Value prompt |

The `/adv-proposal #N` positional syntax is the canonical form (rq-issueChangeLinkage01). It auto-fetches the GH issue body, sanitizes scoring trailers (rq-roadmapOriginSanitize01), prefills the problem statement, and sets `origin_kind: 'roadmap'` + `origin_issue_number: N` on the created change. Agents do NOT need to pass origin args directly when using `#N` ŌĆö the wiring is automatic.

## Phase 3: Quick actions (optional)

If user asks "what should I work on?" or "pick the top item", recommend the highest-WSJF feature without an `active_change`. Do NOT auto-start work ŌĆö keep command read-only. User invokes `/adv-proposal` (or `/adv-atc`) explicitly.

## Coexistence

| Command | Role | Relationship to /adv-roadmap |
|---|---|---|
| `/adv-status` | Operational health: in-flight ADV state, Temporal/worker, worktrees, session debt | Disjoint scope. `/adv-status` answers "is system OK and what am I doing?"; `/adv-roadmap` answers "what's next?". Both surface roadmap freshness (last `/adv-triage` mtime). |
| `/adv-triage` | Regenerates `ROADMAP.md` + `.adv/roadmap-snapshot.json` from GitHub Project | Upstream of `/adv-roadmap`. Without a `/adv-triage` run, `source: 'file'` returns an actionable error. |
| `/adv-proposal` | Starts a new ADV change | Downstream of `/adv-roadmap`. Use `/adv-proposal #N` to auto-link the new change to roadmap item `#N` via `change.origin` (rq-issueChangeLinkage01). |
| `/adv-atc` | Autonomous through-change execution | Downstream of `/adv-roadmap`. Same linkage path applies. |

## Anti-Patterns

| ├Ś Bad | Ō£ō Good |
|---|---|
| Echo only "Top 3 features" without the full requested set | Render the full filtered set inline; truncation only via `--top N` |
| Replace the chat render with "see ROADMAP.md" | Phase 2 mandates inline tables; ROADMAP.md is the canonical mirror, not a substitute for the response |
| Auto-start change ("I'll begin with #51 now") | Read-only command; surface the recommendation, let user invoke `/adv-proposal` |
| Always use `--live` to be safe | File mode is the canonical default; `--live` only when staleness is a concern |
| Skip the active-change cross-reference | The `active_change` annotation is the primary differentiator from raw ROADMAP.md ŌĆö always render it |
| Treat stale file bugs with zero active changes as unstarted work | Render freshness warning; live-check before proposing bug fixes |
