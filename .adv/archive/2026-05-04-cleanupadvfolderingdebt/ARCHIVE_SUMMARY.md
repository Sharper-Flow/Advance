- **[gotcha]** ADV evidence tools can transiently TypeError while recording TDD evidence (`evidenceResult?.task.tdd_phase`, `task.tdd_evidence`), but checkpoint/task update may still persist evidence. When this happens, run the same command via shell, include exact command/exit summary in checkpoint verification, and proceed only after green verification passes.
- **[gotcha]** Use `pnpm exec vitest run <file>` for focused Vitest verification in this repo. `pnpm test -- <file>` can behave like a broader suite run and surface unrelated test failures, which obscures task-local red/green evidence.
- **[pattern]** For safe worktree branch-prefix cleanup, expose a small bounded helper that uses `rmdir` only, stops at canonical base, and throws on namespace escape. Call sites that may receive custom/legacy paths should catch escape and skip cleanup, preserving the primary delete operation while direct helper tests enforce the safety invariant.
- **[pattern]** Archive listing must treat `change.json.id` as canonical identity, not archive directory names. When combining Temporal, source-dir, and archive-dir sources, deduplicate loaded `Change` records by `change.id` and prefer terminal statuses to avoid duplicate archived entries from dated bundle dirs.
- **[pattern]** For external-state file relocations, pass the explicit `ProjectPaths.<file>` path at write call sites, and keep read-only fallback to legacy nested `.adv/<file>` only when the flat override is missing. This preserves compatibility without re-creating nested `.adv` directories.
- **[convention]** Status hygiene findings that imply deletion must be reported as dry-run-only with `deletion_requires_approval: true`. Exclude worker lock lifecycle artifacts from generic cleanup reports when another change owns that behavior.
- **[pattern]** For test-run cleanup of ADV external state, snapshot synthetic project/worktree dirs before Vitest and delete only newly-created synthetic dirs at teardown. Keep real project IDs untouched, and honor `.adv-test-owner` marker mismatches as a no-op guard.
- **[gotcha]** Running Prettier over legacy top-level markdown (`ADV_INSTRUCTIONS.md`, `SETUP.md`, `project.md`) can cause broad unrelated reflow. For docs-refresh tasks, avoid committing formatting-only churn unless the repo check explicitly requires it; keep targeted diffs.
