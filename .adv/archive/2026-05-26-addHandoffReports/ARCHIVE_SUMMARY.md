## Wisdom Accumulated

- **[gotcha]** When the OpenCode session is launched from the canonical checkout, ADV state mutations such as adv_task_update can hit the trunk-write isolation guard even after adv_worktree_resume. Route the mutation with target_path set to the resumed ADV worktree and explicit target confirmation.
- **[pattern]** When introducing new sub-agent report variants, keep the current task-consumer schema (`SupportedSubagentReportSchema` / `SubagentReportSchema`) task-scoped until persistence/readback consumers migrate; expose a separate all-variant schema (`ScopedSubagentReportSchema`) for new optimized handoff validation. This preserves typecheck and legacy task.subagent_reports[] compatibility.
- **[pattern]** For report sidecar migration, preserve legacy task report idempotency keys whenever taskId is present (`changeId|taskId|agent|attempt`) and use scope-aware keys only for taskless/change-scoped reports. This keeps old Temporal histories replay-safe while enabling new sidecar scopes.
- **[pattern]** For taskless report submit, compute source identity from structural scope and omit `taskId` from the signal payload; task-scoped reports keep taskId for legacy consumers. Follow-up agenda descriptions should include change/scope/agent/attempt plus task id when present.
- **[pattern]** During sidecar migrations, opt-in readback should merge canonical sidecar entries with legacy embedded entries and dedupe by the same report key. Compatibility checks such as checkpoint legacy-output suppression must also consult sidecar task-scoped reports, not only task.subagent_reports[].
- **[pattern]** For stale enforcement-doc cleanup, add asset tests against live orchestration docs rather than only searching manually. Keep historical CHANGELOG references allowed, but block active ADV_INSTRUCTIONS/.opencode agent claims and repository maps for retired directories.
- **[pattern]** When adding optimized handoff reports, pin both agent prompt transport and command packet anchors in one asset test. Validate report-specific anchors via getSubagentReportPacketAnchors so schema field-source changes break prompt/packet tests deterministically.
- **[gotcha]** Final `pnpm run check` can fail only at Prettier after targeted tests pass. For new asset tests, run `pnpm run format` before final check/build to avoid a formatting-only verification retry.
- **[gotcha]** Command packet examples for strict report transports should show a complete schema-valid payload skeleton, not only agent/phase/summary fields; otherwise prompt text can drift even when anchors exist.
