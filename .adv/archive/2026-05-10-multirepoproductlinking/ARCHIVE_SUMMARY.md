## Wisdom Accumulated

- **[gotcha]** adv_change_update writes every provided artifact field, including empty strings. During refinement, omit unchanged fields rather than passing empty strings, or previously approved proposal/agreement/design content can be clobbered and must be restored from tool output.
- **[gotcha]** Product-linked wisdom/reflection tools must route Temporal change handles through store.productContext.productProjectId when present. Using repo-local getProjectId(store.paths.root) from a secondary repo can miss the canonical product-state workflow.
- **[pattern]** Archive multi-repo safety should collect and validate every scoped repo before writing bundles or merging. Sort scope_repos by merge_order, capture immutable before/after refs, and fail before side effects when any required repo fails ff-only ancestry preflight.
- **[gotcha]** Integration tests that instantiate AdvancePlugin can exceed Vitest's 5s default under full-suite Temporal worker/bundle load even when they pass individually. Give plugin-init/worker-startup integration tests explicit timeout budgets instead of treating first full-suite timeout as behavior failure.
- **[gotcha]** Product-linked config schemas can expose degradation modes before runtime honors them. Every enum branch like missing_primary_policy must have explicit resolver behavior and tests; otherwise schema-valid configs silently collapse to default blocking behavior.
