ADV's Temporal-backed change workflows can become unreplayable after ADV workflow-code deploys. The current incident shows a real `TMPRL1100` nondeterminism failure: old history scheduled `inspectArtifactActivity`, while current replay expects a different command sequence around search-attribute upserts. Because poisoned workflows stay visible as non-terminal claims, `adv_wip_state` and worktree coordination can fail or hide healthy worktrees.

Desired outcome: ADV treats Temporal workflow logic as a versioned durable protocol. CI/release blocks replay-incompatible workflow changes; workflow-code changes that affect command order use patching/versioning/reset-safe migration; existing poisoned/stale workflows can be recovered with audited tooling; and read paths like `adv_wip_state` isolate poisoned workflows while keeping healthy WIP visible.
