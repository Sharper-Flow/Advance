ADV read tools are paying unnecessary latency because default status/list/show paths eagerly compute full diagnostic, artifact, and workflow state data even when callers need only lightweight orientation. Operators also lack first-class latency telemetry to identify which tool or substep is slow. This change should make common ADV tool calls faster by making read paths view-/include-aware, adding a correctness-safe summary read model where appropriate, and surfacing latency measurements for regression detection and diagnosis.
