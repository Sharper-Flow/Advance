Fixed cross-project `target_path` state routing for per-project OpenCode shards. ADV now has a shard-aware `getExternalRootForProject()` helper and uses it for target project stores, cross-project change creation, and advisory target dependency reads. Sharded create/read/mutation paths now resolve target state under the target project's canonical shard; non-sharded and non-canonical shard sessions keep legacy fallback behavior. Existing shadow records, including the original `fixArchiveTimeout` record under the pokeedge-web shard, are not migrated.
