- **[gotcha]** `adv_task_update` rejects `status: "cancelled"` before any signal mutation. Worktree isolation guard still needs to run before that rejection when guarding all main-checkout status-transition attempts, otherwise enabled guard would not cover direct cancelled attempts. Hook order: resolve task/change â†’ evaluate guard â†’ existing cancellation-policy rejection â†’ Temporal signal path.
- **[success]** For worktree detection regressions, a small real-git temp fixture gives stronger coverage than mocked `rev-parse`: create main checkout + linked worktree, then assert main=false, linked=true, non-git=false against the shared git-session helper.
- **[gotcha]** `lru-cache` with `allowStale: true` returns stale entries immediately on TTL expiry. For probe cache semantics where ordinary TTL expiry should refresh but abort/rejection should fall back to stale, pass `allowStale: false` to `cache.fetch()` while keeping `allowStaleOnFetchAbort/Rejection` and `ignoreFetchAbort` enabled.
- **[pattern]** Safety-critical restart verification can use probe-cache for bounded probes, but success must require non-stale freshness. Use forceRefresh for verification loops and treat stale cached serviceability as diagnostic-only, not proof for restart/reclaim success.
- **[gotcha]** Full `pnpm run format` can reformat earlier task files during a later verification task. If Prettier drift appears late, rerun targeted docs/assets, `pnpm run check`, full `pnpm test`, and `pnpm run build` before checkpoint so the final verification commit captures all style-only normalization safely.
