## Wisdom Accumulated

- **[gotcha]** Epic workflows can be one-per-entity without reviving the retired project shared workflow pattern, but design/tests must make the boundary explicit: no project singleton aggregation, no child workflow orchestration, workflow-safe imports only, and Visibility/query surfaces for cross-change lookup.
- **[pattern]** For cross-change Epic membership lookup, mirror the backlog claim pattern: store a compact optional projection on each child change, index it via a single-value Temporal Keyword (`AdvEpicId`), and query Visibility by project + Epic ID rather than hydrating all changes.
- **[gotcha]** Adding new MCP tools requires updating not only tool implementation/registry but also frozen tool-name snapshots and CLI surface matrix coverage; otherwise asset/bridge contract tests fail even when tool behavior is correct.
- **[gotcha]** When using repo-local `bin/oc-test targeted`, run it from the repo root and pass plugin test filters as `src/...` paths because the wrapper invokes Vitest with `plugin/` as cwd. Passing `plugin/src/...` yields 'No test files found'.
- **[gotcha]** Epic store reads must not auto-start missing Epic workflows: auto-start hides stale child links and missing-Epic recovery bugs. Create should eagerly start, but get/list should return null for missing workflows and mutations should emit typed `epic_not_found`.
- **[pattern]** When adding a new ADV workflow concept, update orchestration allowlists, typed-worker packet guidance, command context-loading instructions, CLI/asset contract tests, and source-of-truth docs together; otherwise runtime behavior and agent contracts drift.
- **[gotcha]** Adding or reordering ADV Temporal search attributes requires updating all pinned test fixtures in `service.test.ts`, `service-reconnect.test.ts`, and `observability.test.ts`; full-suite failures can also surface stale exact spec-version pins and missing rq-* citation coverage after spec bumps.
- **[gotcha]** When adding backward-compatible schemas that make a formerly-required field optional (legacy `change_id` plus new `change_ref`), update hot-path formatters and type guards in the same task; TypeScript will surface old assumptions as `string | undefined`, and public schema artifacts must be regenerated before schema-registry tests pass.
- **[pattern]** For change-workflow projection signals, keep the full contract aligned in one slice: signal payload schema, signal-name constant, client message binding, workflow local binding/handler, pure state helper, message-surface fixture, and search-attribute tests. Existing signalMutation success path already upserts AdvEpicId when state.epic_membership changes.
- **[gotcha]** Same-project Epic link/unlink/move must mutate the child change through Store.changes APIs, not only the Epic workflow. Keeping Epic entry state and child epic_membership projection in sync requires explicit expected identity checks on clear/move; otherwise unlink can erase the wrong projection or move can silently overwrite another Epic membership.
- **[pattern]** Cross-project Epic membership routing should keep the Epic workflow owner-local and route only child projection mutations through `withTargetPathStore`; link/move entries need explicit `changeProjectId`, `repoId`, and `targetPath` refs so the owner Epic can represent product-scope membership without mutating untrusted targets silently.
- **[pattern]** Epic repair/status surface work needs both mutation and presentation paths in one verified slice: member_status mapping in map/compact output, workflow/store membership-status signal, tool registry/allowlist/title surfaces, and targeted tests for repair modes plus CLI bridge contract.
- **[gotcha]** Spec/docs asset tasks can look green before new scope is actually locked: update both .adv/specs JSON and docs/specs markdown, then strengthen asset tests to assert the new rq-* IDs and ADV_INSTRUCTIONS/tool-matrix wording so future changes cannot drift silently.
- **[success]** Epic integrated verification can be bounded but high-confidence by combining the Epic-focused targeted matrix (types, state, workflow, visibility, store, tools, assets, registry), `pnpm run check`, and repo-local `bin/oc-test full`; this catches both focused retrofit regressions and wider repo contract drift.
