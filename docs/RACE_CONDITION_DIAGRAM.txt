┌─────────────────────────────────────────────────────────────────────────────────┐
│ WHY SLASH COMMANDS TRIGGER THIS MORE OFTEN                                     │
└─────────────────────────────────────────────────────────────────────────────────┘

Regular Message:
  User input:        ~100 tokens
  Agent response:    ~500 tokens
  Total:             ~600 tokens per exchange
  
  To reach 100k:     ~167 regular exchanges
                     = ~10+ hours of conversation

Slash Command:
  Command metadata:  ~1000 tokens
  Command body:      ~4000 tokens (366 lines of workflow)
  Agent response:    ~500 tokens
  Total:             ~5500 tokens per command
  
  To reach 100k:     ~18 slash commands
                     = ~5-10 minutes if user runs commands rapidly
  
  ✅ Explains why user sees amnesia RIGHT AFTER /adv-propose
  ✅ Compaction fires 10x more frequently with commands
  ✅ User notices immediately vs. gradual drift

The PROBLEM is not that state is lost — it's that OpenCode's compaction
doesn't include the hook output in the final compaction prompt!


┌─────────────────────────────────────────────────────────────────────────────────┐
│ WORKAROUND: system.transform Hook (Temporary)                                  │
└─────────────────────────────────────────────────────────────────────────────────┘

Instead of relying on the broken compacting hook, use system.transform
which fires on EVERY message (can't be stripped by compaction):

"experimental.chat.system.transform": async (input, output) => {
  if (state.activeChange.id) {
    output.system.push(
      `[CRITICAL] Active ADV change: ${state.activeChange.id}`
    );
  }
}
