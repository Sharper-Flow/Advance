# Backend Language / Runtime Evaluation

## Consider

- Team expertise and production experience.
- Ecosystem fit — domain libraries, framework maturity, package health.
- Performance characteristics — latency, throughput, memory model.
- Deployment model — target platform runtime support.
- Hiring and onboarding — can team grow with this choice?
- Brownfield fit — integration with existing stack and operations.

## Socratic prompts

1. What language do you debug fastest when production is on fire?
2. Does this project need a library that only exists in one ecosystem?
3. If choosing a new language, who mentors team and owns runbooks?
4. What performance constraint would force you away from current runtime?

## Evidence to gather

- Official runtime/framework docs.
- Current maintenance status and release cadence.
- Real production examples in similar problem domain.
- Existing repo compatibility and migration cost.
- Operational tooling: logging, tracing, profiling, debugging.
