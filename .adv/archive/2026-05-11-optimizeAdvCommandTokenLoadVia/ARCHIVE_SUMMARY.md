## Wisdom Accumulated

- **[pattern]** When extracting a command that already has a shared skill, deepen the skill additively and keep command-owned state mutation (for example adv_project_metadata) in the thin command. Verify skill has no ADV mutator calls while command preserves orchestration and fallback tokens.
- **[success]** Dedicated command extraction can hit both targets when command keeps only phase orchestration and exact approval/mutator tokens, while skill holds bucket heuristics/report templates. adv-cleanup reduced from 244L to 120L with skill at 172L and sync check clean.
- **[pattern]** Reflection-style commands extract cleanly by leaving archived-change guard, tool calls, and error handling in command while moving rubrics/templates into skill. Preserve non-blocking archive wording in command because it is workflow-critical.
- **[pattern]** For research-pack commands, keep file-write boundary and exact external tool names in command; move scan rubrics, applicability scoring, and artifact schema to skill. Verification should grep both docs/*-prep.md boundary and Context7/Kagi tool tokens.
- **[pattern]** For clarification commands already near line target, use missing-skill RED check instead of line-count RED. Keep question-tool and gate/agreement boundary tokens in command; move Socratic taxonomy and smells into skill.
- **[pattern]** For audit/scanner commands, command should keep quality-gate thresholds and metadata write because they are execution contract; skill can hold dimension rubrics, finding schema, and report schema. Missing-skill RED works for commands already under line target.
- **[success]** Small commands can still benefit from skill extraction for reusable methodology even when command line count already passes KD8. Use missing-skill RED, preserve mutating tool names in command, and keep command at same or lower line count.
- **[gotcha]** For bulk instruction compression, create contract-token snapshot before edits and compare exact token sequences after. Shell single quotes can break Python f-strings using x['tokens']; use item["tokens"] or avoid nested single quotes in bash -c.
