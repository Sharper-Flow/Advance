The `adv status` CLI reads active-change state by querying each change workflow (`CHANGE_WORKFLOW_QUERY_NAMES.getState`). A Temporal workflow query is only answered by a worker polling that project's task queue. The ADV worker is started per OpenCode session/project, so a project with no open session has no worker, the query times out after 5s, the CLI returns `live:false`, and the launcher (correctly fail-closed per rq-statusCliBridge01) shows only the plain row. Result: `warp-project-launcher` shows no ADV changes for any project that does not currently have an open ADV session (e.g. ~/dev/pokeedge), even though its change workflows are Running in Temporal.

Change workflows already upsert rich Visibility search attributes on every signal: AdvChangeId, AdvChangeTitle, AdvChangeStatus, AdvCurrentGate, AdvLastSignalAt, AdvCreatedAt, AdvAffectedProjects. These are server-side Temporal Visibility data, readable without a worker. The CLI should build active-change rows from Visibility search attributes instead of per-workflow queries, keeping the read live-Temporal-backed and fail-closed when Temporal/Visibility is unavailable, while removing the dependency on a live per-project worker.
