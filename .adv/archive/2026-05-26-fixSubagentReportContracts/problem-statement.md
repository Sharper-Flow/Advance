ADV sub-agent report handling has contract drift across schemas, persisted state, tool validation, and agent prompt packets. Older persisted reports can fail current read schemas because required fields such as `scope_drift` and `required_main_agent_actions` are missing. Acceptance-stage reviewer packets can also instruct workers to submit reports against synthetic task IDs that `adv_subagent_report_submit` rejects by design. The desired outcome is structurally reliable sub-agent report submission and readback: legacy reports normalize safely, invalid anchors diagnose clearly, acceptance-review packets use a supported anchoring model, and schemas, prompts, specs, and tests agree.
